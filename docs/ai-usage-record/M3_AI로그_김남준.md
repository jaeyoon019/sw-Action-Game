### 제출 시점: (AI 로그  #3)

**이번 기간 동안 AI를 어떻게 활용했나요?**
> 유스케이스 다이어그램을 설계하는 과정에서 AI를 적극적으로 활용하였다. 게임에 필요한 기능을 정리하고 액터와 유스케이스 간의 관계를 구성하는 과정에서 도움을 받았다. 또한 GitHub에서 표현 가능한 Mermaid 형식을 찾아 적용하였으며, 최종 구조는 프로젝트 목적과 구현 범위에 맞게 직접 수정하여 완성하였다.

**AI 활용에서 가장 도움이 된 순간은?**
GitHub Markdown에서 Mermaid 문법을 사용하면 별도의 이미지 파일 없이도 유스케이스 다이어그램을 그림 형태로 표현할 수 있다는 점을 알게 된 것이 가장 도움이 되었다. 이를 통해 다이어그램 수정과 버전 관리가 쉬워졌으며, GitHub 저장소 안에서 설계 문서를 효율적으로 관리할 수 있게 되었다.

**AI 활용에서 가장 어렵거나 주의가 필요했던 순간은?**
처음에는 원하는 결과가 UML 유스케이스 다이어그램인지 단순 기능 목록인지 AI가 정확하게 이해하지 못하였다. 따라서 원하는 형식과 사용 환경을 구체적으로 설명하는 것이 중요하다는 점을 알게 되었다. 또한 AI가 제안한 내용이 실제 프로젝트 범위와 일치하는지 반드시 검토하는 과정이 필요하였다.


**AI 없이 직접 해야 했던 작업은 무엇이었나요? 이유는?**
요구사항 정의서에 같은 경우에는 ai사용법이 익숙치 않아 그냥 직접 검색하고 직접 작성하는 식으로 수행하였다. usecase작성에 관한 경우에 초반 프로젝트 범위에 맞는 기능 선정과 최종 유스케이스 구조 수정은 직접 수행하였다. AI가 제안한 기능 중 일부는 실제 구현 계획과 다르거나 필요하지 않은 기능이 포함되어 있었기 때문이다. 따라서 프로젝트의 목적과 구현 가능성을 고려하여 필요한 기능만 선별하고 구조를 재구성하였다.


**다음 기간에는 AI를 어떻게 다르게 활용할 건가요? (AI 로그 #3인 경우: 이번 학기 AI 활용에서 얻은 가장 중요한 교훈)**
AI는 결과를 그대로 사용하는 도구가 아니라 아이디어를 제안하고 작업 효율을 높여주는 보조 도구라는 점을 배웠다.  특히 설계 과정에서는 AI의 답변이 전부 타당하다 싶기도하다가 막상 직접 확인해본다면 구멍이 나있는 부분들이 많아서 사람이 직접 비판적으로 검토하고 프로젝트 상황에 맞게 수정하는 과정이 중요하다는 것을 느꼈다. 앞으로도 AI를 참고 도구로 활용하되 최종 판단과 책임은 개발자가 가져야 한다는 점을 가장 중요한 교훈으로 생각하게 되었다.
